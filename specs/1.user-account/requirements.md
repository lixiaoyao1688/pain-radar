# 个人模块 — 需求规格

## 概述

用户可以注册、登录、退出登录，并在「我的」页查看个人信息。

## 项目信息

- 项目名: pain-radar
- 架构类型: Turborepo monorepo（apps/web + apps/server + packages/{api,db,auth,ui,env,config}）

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-07-04 | v1   | 初始需求 |

## 用户故事

- 作为新用户，我想要用一个简单的英文账号和密码注册，以便快速开始使用。
- 作为老用户，我想要登录后进入自己的痛点库，退出后数据不被他人看到。
- 作为登录用户，我想要在「我的」页查看我的账号信息和使用统计。

## 功能需求

1. [F-001] 注册：纯英文账号（字母开头，字母+数字，3-30 位）+ 密码（≥6 位）注册（对应 PRD 1-1）
2. [F-002] 登录：账号 + 密码登录，登录态持久化（对应 PRD 1-1）
3. [F-003] 退出登录：「我的」页退出按钮，清除会话并跳转登录页（对应 PRD 1-1）
4. [F-004] 查看个人信息：「我的」页展示头像、用户名、捕获痛点统计（对应 PRD 1-2；统计数据由 feature 2 提供，先占位）
5. [F-005] 路由守卫：未登录访问受保护页面（捕获、痛点库、我的）重定向到登录页

## 非功能需求

- 性能: 登录/注册接口响应 < 1s
- 安全: 密码最少 6 位（前端 zod + 服务端 better-auth `minPasswordLength: 6` 双重校验）；会话 cookie httpOnly
- 兼容性: 移动端优先（390px 视口），深色主题

## 验收标准

- [ ] [AC-001] 用合法英文账号+6 位密码注册成功，自动登录并跳转首页
- [ ] [AC-002] 非法账号（含中文/特殊字符/数字开头）或密码 <6 位时前端拦截并提示，不发请求
- [ ] [AC-003] 注册过的账号能登录成功；错误密码登录失败并提示
- [ ] [AC-004] 「我的」页显示当前用户名；点击退出登录后回到登录页，再访问「我的」被重定向到登录页
- [ ] [AC-005] 未登录直接访问 `/`、`/library`、`/profile` 均重定向到 `/login`

## 依赖

- better-auth（packages/auth 已配置 email+password，minPasswordLength 6）
- PostgreSQL（auth 相关模型 User/Session/Account/Verification 已在 prisma schema 中）

## 开放问题

- 无（账号纯英文、密码 ≥6 位、痛点需登录均已确认，2026-07-04）
