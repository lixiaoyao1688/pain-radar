# 痛点模块 — 需求规格

## 概述

将用户日常遇到的抱怨、摩擦记录为痛点并持久化到数据库，支持在痛点库中浏览与搜索。

## 项目信息

- 项目名: pain-radar
- 架构类型: Turborepo monorepo（apps/web + apps/server + packages/{api,db,auth,ui,env,config}）

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-07-04 | v1   | 初始需求 |

## 用户故事

- 作为创业者/产品经理，我想要随手记录遇到的抱怨和不便，以便沉淀为结构化痛点库。
- 作为用户，我想要浏览、搜索已收集的痛点，以便回顾和发现重复出现的问题。

## 功能需求

1. [F-001] 首页捕获：接收用户输入的痛点文本，提交后存储到数据库（对应 PRD 2-1）
2. [F-002] 输入校验：空内容/纯空白不允许提交，给出提示
3. [F-003] 痛点库：按创建时间倒序展示已收集的痛点列表（对应 PRD 2-2）
4. [F-004] 搜索：按关键词对痛点文本做模糊匹配过滤（对应 PRD 2-2）

## 非功能需求

- 性能: 列表接口常规数据量（千级）下响应 < 500ms；搜索输入需防抖，避免每击键一次请求
- 安全: 输入文本长度上限校验（服务端 zod）；无需登录，接口为 publicProcedure（用户已确认匿名公开）
- 兼容性: 移动端优先（390px 视口），遵循现有深色主题设计

## 验收标准

- [ ] [AC-001] 首页输入痛点文本点击「保存这个痛点」，刷新后在痛点库可见该记录
- [ ] [AC-002] 提交空白内容时不产生数据库记录，且有 toast 提示
- [ ] [AC-003] 痛点库列表按时间倒序展示真实数据库数据（不再是 mock）
- [ ] [AC-004] 在搜索框输入关键词，列表实时过滤为匹配项；清空关键词恢复全量
- [ ] [AC-005] 库为空时展示空状态提示，不报错

## 依赖

- PostgreSQL（packages/db 已配置 Prisma datasource）
- tRPC + Hono（apps/server 已就绪）
