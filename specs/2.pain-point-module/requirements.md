# 痛点模块 — 需求规格

## 概述

登录用户将日常遇到的抱怨、摩擦记录为痛点并持久化，支持在痛点库中浏览与搜索自己的痛点。

## 项目信息

- 项目名: pain-radar
- 架构类型: Turborepo monorepo（apps/web + apps/server + packages/{api,db,auth,ui,env,config}）

## 需求版本

| 日期       | 版本 | 说明                                       |
| ---------- | ---- | ------------------------------------------ |
| 2026-07-04 | v1   | 初始需求（痛点按用户隔离，需登录，已确认） |

## 用户故事

- 作为登录用户，我想要随手记录遇到的抱怨和不便，以便沉淀为我自己的痛点库。
- 作为登录用户，我想要浏览、搜索我收集的痛点，以便回顾和发现重复出现的问题。

## 功能需求

1. [F-001] 首页捕获：接收用户输入的痛点文本，提交后关联当前用户存储到数据库（对应 PRD 2-1）
2. [F-002] 输入校验：空内容/纯空白不允许提交，给出提示；文本长度上限 2000
3. [F-003] 痛点库：按创建时间倒序展示当前用户的痛点列表（对应 PRD 2-2）
4. [F-004] 搜索：按关键词对当前用户的痛点文本模糊匹配过滤（对应 PRD 2-2）
5. [F-005] 统计：「我的」页展示当前用户捕获痛点总数；痛点库「总记录」卡片展示真实数量

## 非功能需求

- 性能: 列表接口千级数据响应 < 500ms；搜索输入 300ms 防抖
- 安全: 接口全部 protectedProcedure，服务端按 session userId 过滤，杜绝越权读写；服务端 zod 校验
- 兼容性: 移动端优先（390px 视口），深色主题

## 验收标准

- [ ] [AC-001] 登录后在首页输入痛点并保存，痛点库可见该记录且归属当前用户
- [ ] [AC-002] 提交空白内容不产生记录且有提示；超长文本被服务端拒绝
- [ ] [AC-003] 用户 A 无法看到用户 B 的痛点（换账号验证）
- [ ] [AC-004] 搜索关键词实时过滤列表，清空恢复全量
- [ ] [AC-005] 新用户痛点库为空时展示空状态；「我的」页统计为 0
- [ ] [AC-006] 未登录调用痛点接口返回未授权错误
